# Implementation Plan: profile-image

## TDD 実装方針（`AGENTS.md` 準拠）

- `/kiro/spec-impl` では **テスト駆動** とし、各実装単位で **Test List（`it.todo()`）→ 1 ケースを `it()` にして Red → Green → Refactor** を繰り返す。Test List は「計画」であり、実装中にケースを追加・削除してよい。
- **ケース列挙とテストコードの記述は分ける。** 先に「何を検証するか」だけを列挙（`it.todo()` のラベルやチェックリストにとどめ、アサーションやセットアップの本実装は書かない）。列挙が一段落してから、ケースを 1 つずつ `it()` に落として Red を取る。ケースとコードを同時に書くと過剰なケースが増えやすいため、この二段階を徹底する。
- 本一覧では **`/kiro/spec-tasks` のルールどおり、対応するテストタスクを実装タスクの直前**に配置する。
- **例外**（テスト前置なしの単独タスク）: スキーマ拡張・マイグレーション、依存追加と環境・バケット前提、ランタイムの画像ホスト許可のみの設定など、テストが本質的に先に書けない／不要なもの。

テストレイヤー・モック方針の技術詳細は、プロジェクトのテスト実装ガイド（単体→結合→E2E の判断、実 DB、`.error.test.ts`、Storybook と `pnpm test` の分離）に従う。
